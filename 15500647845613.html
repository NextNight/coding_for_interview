时间复杂度与空间复杂度

时间复杂度和空间复杂度是衡量算法性能好还的标准，在代码为执行前进行分析可以得到时间和空间消耗的预估，而一个好的算法必然能在二者之间得到平衡。复杂度通常书取决于执行次数最多的代码。

时间复杂度的几种类型

def line_search(nums,key):
    for i,num in enumerate(nums):
        if num==key:
            return i
    return None
  • 最好，最坏时间复杂度：两种极端情况，代码最少执行的次数和代码最多次数。
  • 加权平均复杂度：当一段代码执行次数的可能性有多种的时候，代码复杂度不能用最好最坏这种极端的情况来表示，此时可以用每种可能性的概率*次数在求和/可能性的个数 就可以得到加权平均复杂度。

以上的line_search最好的时间复杂度是：O(1),最坏是：O（N）,但是每个nums数组的每个索引上都有可能知道key，同时也可能所有位置都找不到，假设：找到找不到的概率分别是0.5，那么加权平均复杂度就是：0.5($\sum _{ i }{ N }{ 1{ p }_{ i } }$)+0.5

  • 均摊时间复杂度：均摊时间复杂度指的是在某种情况下会偶尔出现复杂度较高的情况，大部分时间都是较低的复杂度，此时可以将这个较高的复杂度均摊到这些复杂度较低的步骤上，这样复杂度均摊之后就近似于低复杂度的情况：

时间复杂度的常见量级

多项式量级

  • O(1):常数量级，不管是1，2，3还是更多次，只要是固定的不随数据量n而改变的就常数量级的复杂度
  • O(n):线性量级,复杂度与数据量n成线性关系，无论是(O(a*n)，a为常数，统称为O(n))
  • O(n2):K次方介，复杂度是数据量的k次方
  • O(nlogn)：线性多数介
  • O(logn)：对数介，不论底数是2，3，4，5。。。都表示为logn,对数可以使用换底公式都转换成常数*logn的形式。

非多项式量级

以下两种量级的复杂度，随数据量的增长急剧上升，截至目前没有对这类问题有一个好的解决方案，又称为NP问题。

  • O(2n):指数介
  • O(n!):阶乘介
